# Checklist — Adding a New Stage

Turnkey numbered checklist for adding one of the 8 selectable stages (or the intro stage) end to end. Work top to bottom; each step's playtest gate must pass before moving to full-stage playtest at the bottom.

1. **Create the `StageData` resource** (`.tres`) in `references/godot/templates/` conventions — set stage id, display name, weakness-cycle archetype (see `references/design` for the Frost/Flame/Storm/Volt/Stone/Toxin/Blade/Gravity cycle), music track reference, boss reference, and the mid-boss reference if this stage has one.
2. **Build the tilemap scene.** Use `TileMapLayer` (not the deprecated single `TileMap` node — see `gotchas.md` #12) at the 256x224 internal resolution grid. Block out the critical path first (start -> mid-boss gate if any -> boss door), then decorate.
3. **Configure the stage gimmick.** Every stage should have exactly one signature environmental mechanic (conveyor belts, disappearing blocks, wind currents, ice-slide floors, vertical climb sections, etc.) tied to that stage's weakness archetype mood (see `assets/audio-spec.md` mood-matching guidance). Wire the gimmick as a reusable node/script, not a one-off hack specific to this stage, so it can be reused or referenced later.
4. **Build the enemy spawn table.** List every regular enemy placement with position, patrol/behavior params, and (if relevant) a gimmick-interaction flag (e.g., "this turret is disabled by the stage's electrical gimmick when active"). Keep small-enemy density sane per `assets/spritesheet-spec.md` sizing/budget guidance — dense fields of 16x16 fodder are cheap to render but must still be individually fair.
5. **Place checkpoints.** Distribute checkpoints so no two consecutive checkpoints (including stage-start and the boss-door checkpoint) are more than ~90 seconds apart on a competent-but-not-perfect playthrough. Err toward more checkpoints early in a stage's critical path where gimmicks/enemies are still being taught (see gate below).
6. **Place the mid-boss** (if this stage has one) at roughly the stage's midpoint, gated behind a checkpoint on both sides. Reference its `BossData` per `checklists/new-boss.md` — a mid-boss still needs its own intro beat and telegraph-clean attack set, just scaled down from a full boss fight.
7. **Build the boss-door ritual.** This is the short pre-boss sequence: a door/gate object, an approach corridor (often gimmick-free, a "breather" beat), and the trigger that locks the door behind the player and starts the boss intro sequence (see `checklists/new-boss.md` step 3). Confirm the checkpoint placed just before this door survives a death-and-retry loop without re-forcing a long walk back.
8. **Hook music and jingles.** Assign the stage theme (mood-matched per `assets/audio-spec.md`) to the stage's ambient music bus on scene load; confirm the checkpoint jingle, death jingle, and boss-intro sting are wired to their respective trigger points (checkpoint touch, player death, boss-door trigger) rather than left on generic defaults.
9. **Wire WorldFlags reads and writes.** List every flag this stage **reads** (e.g., "has the player already obtained the Frost weapon, unlocking the ice-gimmick shortcut") and every flag this stage **writes** (e.g., "mid-boss defeated," "secret item collected," "stage cleared"). Cross-check this list against other stages/systems that reference the same flags — a stage that writes a flag nothing reads, or reads a flag nothing ever writes, is a sign of a missing or dead connection.
10. **Confirm weakness-chart placement.** This stage's boss must occupy exactly one position in the 8-archetype weakness cycle (see `checklists/new-weapon.md` for the full cycle-integrity rule) — verify against the current chart that no archetype is duplicated or missing before treating this stage as content-complete.
11. **Playtest gate — buster-only beatable.** A player with no unlocked weapons (buster only) must be able to complete this stage, mid-boss, and reach/survive the boss door, using only jump/dash/buster. If any gimmick or enemy placement assumes a specific unlocked weapon, either redesign it or explicitly gate that section behind the appropriate WorldFlag so it's unreachable until the player has the prerequisite weapon.
12. **Playtest gate — checkpoint spacing.** Time an actual playthrough death-to-death; confirm no checkpoint gap exceeds ~90 seconds. If a gap fails this, add a checkpoint rather than shortening the section — checkpoint density is cheap, section redesign is not.
13. **Playtest gate — gimmick taught before tested lethally.** Walk the stage's critical path in order and confirm the first encounter with this stage's gimmick is low/no-stakes (a safe demonstration — e.g., a conveyor belt in an open area with no pit) before any encounter where the same gimmick can kill or seriously punish the player (e.g., a conveyor belt feeding into a pit or hazard). A gimmick's first appearance must never be its most punishing appearance.
